## FASE 2 — CORREÇÃO DOS PROBLEMAS DE BUILD E DEPLOY

### T2.1 — Resolver race condition do frontend (CAUSA RAIZ DO BUILD QUEBRADO)

**Problema confirmado:** `yarn build` roda em background enquanto `php -S` já serve
requisições. Resultado: assets ausentes nas primeiras requisições.

**Solução — modificar `start.sh` para build síncrono antes do servidor:**

```bash
# ANTES de iniciar o PHP server, garantir que o build está completo
if [ ! -f public/build/entrypoints.json ]; then
    echo "Building frontend assets (synchronous)..."
    yarn build 2>&1 | tee /tmp/yarn_build.log
    if [ $? -ne 0 ]; then
        echo "❌ Frontend build FALHOU. Ver /tmp/yarn_build.log"
        exit 1
    fi
    echo "✅ Frontend build concluído"
fi
```

Remover o bloco atual que faz `yarn build ... &` em background.

**Verificação pós-mudança:**
```bash
ls -la public/build/entrypoints.json
curl -s -o /dev/null -w "%{http_code}" http://localhost:5000/
```

### T2.2 — Resolver timezone MySQL (Gap #4)

Adicionar no `start.sh`, após o bloco "MySQL is ready", antes do JWT:

```bash
# Align MySQL timezone with PHP runtime
mysql -u root --socket=/home/runner/mysql_run/mysql.sock \
  -e "SET GLOBAL time_zone = 'America/Sao_Paulo';" 2>/dev/null || true
echo "MySQL timezone alinhada: America/Sao_Paulo"
```

**Verificação:**
```bash
mysql -u chamilo -pchamilo_pass --socket=/home/runner/mysql_run/mysql.sock \
  -e "SELECT NOW(), @@global.time_zone, @@session.time_zone;"
php -r "echo date('Y-m-d H:i:s');"
# Os dois devem mostrar o mesmo horário
```
