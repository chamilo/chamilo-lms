Stack de Servidor:

 PHP 7.4 instalado (versão mínima para Chamilo 1.11 — atenção: PHP 8.0+ é incompatível sem patches específicos)
 Extensões obrigatórias presentes: curl, dom, fileinfo, intl, gd, xml, mbstring, zip, mysql/mysqli, pdo_mysql
 Apache com mod_rewrite habilitado ou regras de redirect equivalentes configuradas no servidor do Replit
 Arquivo .htaccess na raiz do Chamilo presente e com AllowOverride All (obrigatório desde v1.10 — sua ausência quebra todo o sistema de URLs)

Configurações php.ini:

 file_uploads = On
 allow_url_fopen = On
 short_open_tag = On
 memory_limit = 256M
 upload_max_filesize = 100M
 post_max_size = 100M
 max_execution_time = 300
 date.timezone = America/Sao_Paulo (ou fuso correto)

Banco de Dados — criação e verificação:

 MySQL 5.6+ ou MariaDB 5.5+ rodando e acessível
 Banco criado com charset correto:

sql  CREATE DATABASE chamilo CHARACTER SET utf8 COLLATE utf8_general_ci;
  CREATE USER 'chamilo_user'@'localhost' IDENTIFIED BY 'SENHA_FORTE';
  GRANT ALL PRIVILEGES ON chamilo.* TO 'chamilo_user'@'localhost';
  FLUSH PRIVILEGES;

⚠️ O usuário precisa ter permissão DROP. Sem ela, a instalação falha silenciosamente ao tentar limpar resíduos de outros sistemas.


 Conexão testada: host, porta 3306, usuário, senha e nome do banco confirmados
 Tabelas principais criadas pelo wizard: user, course, session, access_url (confirme existência)
 Sem tabelas com prefixo de outra instalação (se houver, execute DROP TABLE nelas)

Status da Instalação Web:

 Wizard executado com sucesso via /main/install/
 Arquivo configuration.php presente em main/inc/conf/ com $c_hostname, $c_username, $c_password, $c_database, $c_table_prefix = ''
 Pasta /main/install/ removida ou protegida após conclusão (nunca remover antes de confirmar 100% funcional)
 Permissões de escrita confirmadas em: app/, main/upload/, archive/, home/, certificates/
 Login do admin funcional

2.1 — ARCHITECTURE.md
markdown# Chamilo LMS — Arquitetura do Projeto

## Estrutura de Diretórios
| Pasta | Função | Política |
|---|---|---|
| /main/ | Core da aplicação | NÃO modificar sem versionamento do diff |
| /app/ | Dados dinâmicos (cache, uploads, logs, config) | Não versionar (exceto config template) |
| /vendor/ | Dependências Composer | Não versionar |
| /web/ | Assets públicos (CSS, JS, imagens) | Customizar apenas em subpastas próprias |
| /plugin/ | Plugins e extensões | Zona segura — desenvolver aqui |
| /main/template/ | Templates de UI | Customizar com sufixo _custom |

## Banco de Dados
- Motor: MySQL/MariaDB | Charset: utf8
- Schema: banco único desde v1.9 (sem prefixo de múltiplos DBs)
- Convenção de nomenclatura: prefixo `c_` nas tabelas do core
- Migrações: via upgrade scripts em /main/install/ — nunca alterar manualmente

## Pontos de Extensão Seguros (sem conflito com upstream)
- Plugins próprios: /plugin/[nome-do-plugin]/
- Temas: /web/css/themes/[meu-tema]/
- Idiomas/subidiomas: /main/locale/
- Hooks/eventos: sistema nativo de eventos do Chamilo

## Zonas de RISCO (evitar alterações diretas)
- /main/inc/ → código core
- /main/install/ → scripts de instalação e upgrade
- /vendor/ → nunca editar dependências

## Compatibilidade
- PHP: 7.4 recomendado (8.0+ requer patches)
- MySQL: 5.6+ / MariaDB: 5.5+
- Apache: mod_rewrite obrigatório (ou rewrite rules em Nginx)

🔧 FASE 3 — CONFIGURAÇÃO DE AMBIENTE REPLIT
3.1 — Secrets (nunca em código)
Configure no painel Secrets do Replit:
DB_HOST=localhost
DB_PORT=3306
DB_NAME=chamilo
DB_USER=chamilo_user
DB_PASSWORD=[senha segura]
CHAMILO_ADMIN_EMAIL=[email do admin]
APP_ENV=development

⚠️ Secrets do Replit não persistem em Forks — documente quais precisam ser reconfiguradas.

Leia em configuration.php via getenv() em vez de hardcoded:
php$_configuration['db_host'] = getenv('DB_HOST');
$_configuration['db_user'] = getenv('DB_USER');
// etc.
