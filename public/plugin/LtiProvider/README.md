# LTI Provider Plugin

Version 2.x

This plugin adds **LTI 1.3 Advantage provider** capabilities to Chamilo 2.

With this plugin, Chamilo can act as an **LTI tool/provider** and expose selected learning content so it can be launched from another LMS or external platform.

At the moment, the provider flow is intended to work with Chamilo activities such as:

- **Exercises**
- **Learning paths / lessons**

In this setup:

- **Chamilo** acts as the **LTI provider/tool**
- the external LMS acts as the **platform/consumer**

---

## What this plugin does

Once installed and enabled, the plugin allows administrators to:

1. Access the **LTI 1.3 Advantage Provider** administration page
2. Review the provider endpoints generated by Chamilo
3. Register one or more external LMS platforms
4. Associate each external platform with a specific Chamilo activity
5. Allow remote LTI 1.3 launches into that activity

This means a remote LMS can launch selected Chamilo content directly through the LTI 1.3 Advantage flow.

---

## Installation and activation

In Chamilo 2, the plugin lifecycle is managed from the **Plugins** administration page.

### Typical steps

1. Go to **Administration > Configuration settings > Plugins**
2. Install the plugin
3. Enable the plugin
4. Open the plugin administration page:
    - **Administration > LTI 1.3 Advantage Provider**
5. Review the provider connection details
6. Register the external LMS platform(s)

> No manual database table creation is required.
> No manual SQL steps are needed in the README for Chamilo 2.

---

## Current administration flow

After enabling the plugin, the main administration page provides two main actions:

- **LTI connection details**
- **Add a customer platform**

### LTI connection details

This dialog shows the provider endpoints generated by Chamilo.

These values must be copied into the external LMS when registering Chamilo as an LTI 1.3 tool.

The dialog exposes values such as:

- **Launch URL**
- **Login URL**
- **Redirect URL**
- **Keyset URL (JWKS)**

### Add a customer platform

This form is used to register an external LMS/platform that will connect to Chamilo.

Each registration links one consumer platform to one Chamilo provider tool/activity.

## Example provider connection details

The following example uses placeholders. Replace them with the real values of your environment.

| Field | Example value |
|---|---|
| Launch URL | `<CHAMILO_BASE_URL>/plugin/ImsLti/launch.php` |
| Login URL | `<CHAMILO_BASE_URL>/plugin/ImsLti/auth.php` |
| Redirect URL | `<CHAMILO_BASE_URL>/plugin/ImsLti/redirect.php` |
| Keyset URL (JWKS) | `<CHAMILO_BASE_URL>/plugin/ImsLti/jwks.php` |

Example:

```text
<CHAMILO_BASE_URL> = https://your-chamilo.example.com
```

Then the provider endpoints would look like:

```text
Launch URL:   https://your-chamilo.example.com/plugin/ImsLti/launch.php
Login URL:    https://your-chamilo.example.com/plugin/ImsLti/auth.php
Redirect URL: https://your-chamilo.example.com/plugin/ImsLti/redirect.php
Keyset URL:   https://your-chamilo.example.com/plugin/ImsLti/jwks.php
```

> Adjust the endpoint paths if your current implementation exposes different routes.

## Customer platform registration fields

When you create or edit a platform registration, the following fields are available.

### LMS (issuer)
Unique issuer identifier of the external LMS/platform.

Example:

```text
<REMOTE_PLATFORM_ISSUER>
```

For example:

```text
https://moodle.example.com
```

### OIDC Auth URL
Authorization/login initiation URL of the external LMS.

Example:

```text
<REMOTE_PLATFORM_OIDC_AUTH_URL>
```

For example:

```text
https://moodle.example.com/mod/lti/auth.php
```

### OAuth2 Access Token URL
Token endpoint of the external LMS.

Example:

```text
<REMOTE_PLATFORM_TOKEN_URL>
```

For example:

```text
https://moodle.example.com/mod/lti/token.php
```

### Keyset URL
JWKS endpoint of the external LMS.

Chamilo uses it to validate signed messages received from that platform.

Example:

```text
<REMOTE_PLATFORM_JWKS_URL>
```

For example:

```text
https://moodle.example.com/mod/lti/certs.php
```

### Client ID
Client identifier assigned by the external LMS to this Chamilo tool registration.

Example:

```text
<REMOTE_PLATFORM_CLIENT_ID>
```

### Deployment ID
Deployment identifier provided by the external LMS.

Example:

```text
<REMOTE_PLATFORM_DEPLOYMENT_ID>
```

### Key id (kid)
Key identifier associated with the registration keys.

Example:

```text
<REMOTE_PLATFORM_KID>
```

### ToolProvider
Defines the type of Chamilo activity exposed through this registration.

Current examples include:

- Exercises
- Learning paths

### Provider tool
Defines the exact Chamilo activity that will be launched.

For example:

- a specific exercise
- a specific learning path / lesson

## Actual configuration model in Chamilo 2

For the current Chamilo 2 implementation, there are two levels of setup:

### 1. Plugin state
Managed from the main **Plugins** page:

- Install
- Enable
- Disable
- Uninstall

### 2. Functional LTI provider setup
Managed from the plugin administration page:

- view Chamilo provider connection details
- add or edit customer platform registrations
- choose the provider tool type
- choose the specific activity to launch

The operational configuration is managed through the Chamilo administration UI and the plugin admin pages.

---

## Typical integration workflow

A typical setup looks like this:

1. Install and enable the plugin from the **Plugins** page
2. Open **Administration > LTI 1.3 Advantage Provider**
3. Open **LTI connection details**
4. Copy the Chamilo provider endpoints:
    - Launch URL
    - Login URL
    - Redirect URL
    - Keyset URL
5. In the external LMS, create the LTI 1.3 tool registration for Chamilo
6. Copy the platform values from the external LMS into Chamilo:
    - LMS (issuer)
    - OIDC Auth URL
    - OAuth2 Access Token URL
    - Keyset URL
    - Client ID
    - Deployment ID
    - Key id (kid)
7. Select the tool type
8. Select the exact provider tool/activity
9. Save the platform registration
10. Launch the activity from the external LMS

---

## Notes

- This plugin is intended for **LTI 1.3 Advantage**
- The exact activities available depend on the current Chamilo implementation
- Values must match exactly between Chamilo and the consumer platform
- If a launch fails, first verify:
    - issuer
    - client ID
    - deployment ID
    - OIDC endpoints
    - JWKS endpoints
    - selected provider tool

---

## Summary

This plugin turns Chamilo 2 into an **LTI 1.3 provider** for selected activities.

It allows administrators to:

- enable the provider integration from the plugin manager,
- retrieve Chamilo provider endpoints,
- register consumer LMS platforms,
- and associate each registration with a specific Chamilo activity such as an exercise or a learning path.
